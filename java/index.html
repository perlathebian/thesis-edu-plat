Introduction to Java

Java is a high-level, object-oriented programming language that was designed with a few key principles in mind: platform independence, simplicity, and security.

Key Concepts:

  • Platform Independence: Java programs can run on any device that has the Java Virtual Machine (JVM), making it a "write once, run anywhere" language.

    For detailed information on Java's platform independence, visit this page.

  • Object-Oriented: Java follows the object-oriented programming (OOP) paradigm, emphasizing the use of classes and objects for modular and efficient code.
  • Syntax and Structure: Java syntax is similar to other C-based languages, making it accessible to developers familiar with languages like C++ or C#.
  • Memory Management: Java handles memory management automatically through its garbage collection mechanism, reducing the risk of memory leaks.
  • Security: Java incorporates built-in security features, such as the sandbox model, to protect against malicious activities.

Basic Components:

  • Classes and Objects: Java is designed around the concept of classes and objects, allowing developers to structure code in a modular and reusable manner.
  • Variables and Data Types: Java supports various data types, including primitive types (int, float, char) and reference types (objects).
  • Control Flow: Java provides constructs for decision-making (if, else statements), looping (for, while loops), and branching.
  • Methods: Functions in Java are called methods and are defined within classes. They allow for code organization and reuse.

Data Types and Variables

Java Variables

Variables are named storage locations that hold data, and they can be declared using a specific data type. A variable declaration in Java typically looks like:

dataType variableName;

In Java, there are different types of variables, for example:

  • String - stores text, such as ‚ÄúHello‚Äù. String values should be wrapped inside double quotes
  • int - stores integers (whole numbers), without decimals, such as 63 or -128.
  • float - stores floating point numbers, with decimals, such as -54.33 or 54.33
  • char - stores single characters, such as ‚Äòb‚Äô or ‚ÄòB‚Äô. Char values are surrounded by single quotes
  • boolean - stores values with two states: true or false

Declaring a Variable

To create a variable, you must specify the type and assign it a value using the following syntax:

dataType variableName = value;

Example:


                    String name = "Joe"; //creating a variable of type String and assigning it the value "Joe"
                    int number = 17; //creating a variable of type int and assigning it the value 15

You can also re-assign a value to an existing variable, it will overwrite the previous value:


                    int number = 17;
            number = 89; //new value is now 89
            /*If you don't want others or yourself to overwrite an exisitng value
            you can use the final keyword; this will declare the value as "final" or
            "constant", which means immutable(unchangeable) and read-only*/
            final int number = 17;
            number = 89; // this will generate an error: cannot assign a value to a final variable

You can try for yourself displaying variables using the println()method, and concatenating different variables (using the plus sign +). Note: concatenating or adding different variable types might generate errors depending on the types used. (example: you cannot add an integer to a text).

Naming Variables:

Variable names are called identifiers. They should always start with lower-case letters and can contain letters, digits, underscores, and dollar signs. Names are case-sensitive (myNum is different from mynum). Reserved words like Java keywords (such as int or boolean) cannot be used as identifiers/names.

Data Types

Data types are divided into two groups:

  • Primitive data types - include byte, short, int, long, float, double, boolean, and char.
    • A primitive data type specifies the size and type of variable values, and it has no additional methods.
  • Non-primitive data types - such as String, Arrays, and Classes (more on these later)
    • Non-primitive data types are called reference types because they refer to objects.

The main difference between primitive and non-primitive data types are:

  • Primitive types are predefined (already defined) in Java. Non-primitive types are created by the programmer and is not defined by Java (except for String).
  • Non-primitive types can be used to call methods to perform certain operations, while primitive types cannot.
  • A primitive type has always a value, while non-primitive types can be null.
  • A primitive type starts with a lowercase letter, while non-primitive types starts with an uppercase letter.

Type Casting

Type casting refers to the process of converting a variable from one data type to another, allowing the variable to be used in expressions or operations that require a different type. In Java, there are two types of casting:

  • Widening casting (automatic) - converts a smaller type to a larger type size: byte ‚Üí short ‚Üí char ‚Üí int ‚Üí long ‚Üí float ‚Üí double
  • Narrowing casting (manual) - converts a larger type to a smaller type size: double ‚Üí float ‚Üí long ‚Üí int ‚Üí char ‚Üí short ‚Üí byte

                    //example of widening casting

            System.out.println(myDouble); //output 9.0
            
            //example of narrowing casting
            double myDouble = 5.14d;
            int myInt = int(myDouble); //manual casting: double to 
                int
            System.out.println(myDouble); //output: 5.14
            System.out.println(myInt); //output: 5

Java Conditions and If Statements

Java has four condition statements that are used to perform different decisions after testing for different actions.

Loops

Loops are control flow structures in programming that allow a certain block of code to be executed repeatedly based on a specified condition. They help automate repetitive tasks and streamline the execution of code. There are different types of loops, including:

  • For loop: executes a block of code a specific number of times.
    
                for (int i = 0; i < 5; i++) {
                    // Code to be repeated
                }
  • For-Each loop: used exclusively to loop through elements in an array:
    
                    //syntax
                    for (type variableName : arrayName) {
                    // code block to be executed
                    }
                
    
                //example
                String[] cars = {"Volvo", "BMW", "Ford", "Mazda"};
                for (String i : cars) {
                    System.out.println(i);
                }
  • While loop: repeats a block of code as long as a given condition is true.
    
                    while (condition) {
                        // Code to be repeated
                    }
  • Do-While loop: similar to the while loop, but the condition is checked after the code block is executed, ensuring that the block is executed at least once. 
    
                    do {
                        // Code to be repeated
                    } while (condition);
  • It is also possible to place a loop inside another loop. This is called a nested loop. The "inner loop" will be executed one time for each iteration of the "outer loop":
    
                // Outer loop
                for (int i = 1; i <= 2; i++) {
                System.out.println("Outer: " + i);// Executes 2 times
                
                // Inner loop
                for (int j = 1; j <= 3; j++) {
                    System.out.println("Inner: " + j); // Executes 6 times (2 * 3)
                }
                }

Functions and Methods

Exceptions

Exceptions are events that occur during the execution of a program that disrupts the normal flow of instructions. Different errors can occur: coding errors made by the programmer, errors due to wrong input, or other unforeseeable things.

When an error occurs, Java will normally stop and generate an error message. The technical term for this is: Java will throw an exception(throw an error). Exception handling in Java is done using try, catch, and finally blocks.

  • try block: It contains the code that might throw an exception.
  • catch block: It contains the code that is executed if an exception occurs in the corresponding try block. Multiple catch blocks can be used to handle different types of exceptions.
  • finally block: It contains the code that is always executed, regardless of whether an exception occurs or not. It's typically used for cleanup tasks.

Here's the syntax:


                try {
                    // Code that might throw an exception
                } catch (ExceptionType1 e1) {
                    // Code to handle ExceptionType1
                } catch (ExceptionType2 e2) {
                    // Code to handle ExceptionType2
                } finally {
                    // Code that always executes
                }

Example:


                public class ExceptionExample {
                    public static void main(String[] args) {
                        try {
                            // Code that might throw an exception
                            int result = divide(10, 0);
                            System.out.println("Result: " + result);
                        } catch (ArithmeticException e) {
                            // Code to handle ArithmeticException
                            System.out.println("Error: Cannot divide by zero.");
                        } finally {
                            // Code that always executes
                            System.out.println("Finally block executed.");
                        }
                    }
                
                    // A method that may throw an exception
                    public static int divide(int a, int b) {
                        return a / b;
                    }
                }

In this example, the divide method attempts to perform a division operation that may result in an ArithmeticException. The try block is used to enclose this code, and the catch block handles the specific exception type. The finally block always executes, whether an exception occurs or not.

OOP Concepts

Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around the concept of "objects", which represents real-world entities and encapsulate data and behavior. Key principles of OOP include encapsulation, inheritance, and polymorphism.

Encapsulation involves bundling data (attributes) and methods (functions) that operate on the data into a single unit (object).

Inheritance allows objects to inherit properties and behaviors from other objects, promoting code reuse.

Polymorphism allows objects of different types to be treated as objects of a common type, providing flexibility and extensibility in code design.

OOP promotes modularity, reusability, and a more intuitive way of modeling complex systems.

Classes and objects are two main aspects of object-oriented programming. For example, class Fruit could have objects like Apple, Banana, Mango. Another example, class Car could have objects like Volvo, Audi, and Toyota. So, a class is a template for objects, and an object is an instance of a class. When the individual objects are created, they inherit all the variables and methods from the class.

Everything in Java is associated with classes and objects, along with its attributes and methods. For example: in real life, a car is an object. The car has attributes, such as weight and color, and methods, such as drive and brake. A class is like an object constructor, or a "blueprint" for creating objects. 


            //creating a class called Car with member variables

            /*creating an object from the class; specify the class name,

            public static void main(String[] args){
                Car car1 = new Car();
                car1.color = "red";
                car1.speed = "100";

The term variable or member variable refers to the attribute or member field of the class. In the above example, the Car class has attributes color, speed, model. You can access attributes by creating an object of the class (car1), and using the dot syntax (.). This way you can also assign values to the attributes. If you don't want the ability to override existing values, declare the attribute as final.

Constructors

A constructor is a special method that is used to initialize objects. The constructor is called when an object of a class is created. It can be used to set initial values for object attributes:


            //create a Car class
            public class Car{
                int x; //create a class attribute
                
                //create a class constructor for the Car class
                public Main(){
                    x = 5; //set the initial value for the class attribute x
                }
                
                public static void main(String[] args){
                    Car car = new Car(); //create an object of class Car
                    System.out.println(car.x); //print the value of x
                }
            }
Note that you cannot create another constructor in the same class with the same parameters.

Constructors can also take parameters, which is used to initialize attributes. 

 
            public class Car{
                int x;
                
                public Car(int y){
                    x = y;
                }
                
                public static void main(String[] args){
                    Car car = new Car(5);
                    System.out.println(car.x);
                }
            }

This example adds an int y parameter to the constructor. Inside the constructor we set x to y. When we call the constructor, we pass a parameter to the constructor (5), which will set the value of x to 5.

Access Modifiers

For classes, you can either use public (class would be accessible by any other class) or default (class is only accessible in the same package). For attributes, methods, and constructors, you can either use public (code accessible to all classes), private (code is only accessible within the declared class), default (code is accessible in the same package), or protected (code is accessible in the package and subclasses).

Non-Access Modifiers

For classes you can either use final (class cannot be inherited by other classes), or abstract (class cannot be used to create objects, to access the class, it must inherited from another class). For attributes and methods, you can use final (attributes and methods cannot be overridden/modified), static (attributes and methods belong to the class rather than an object), abstract (can only be used in an abstract class, and can only be used on methods; the method does not have a body, the body will be provided by the subclass it inherited from ), transient (attributes and methods are skipped when serializing the object containing them), synchronized (methods can only be accessed by one thread at a time), or volatile (value of an attribute is not cached thread-locally, and is always read from the "main memory").

This concept makes sure that sensitive data is hidden from users. To achieve this, you must declare member variables as private them provide public get and set methods to access and update the values of those member variables.

Inheritance

This involves inheriting attributes and methods from one class to another, thus having a subclass (child) inheriting from a superclass (parent). To inherit from a class, use the keyword extends.

In the example below, the Car class (subclass) inherits the attributes and methods from the Vehicle class (superclass):


            class Vehicle {
            protected String brand = "Ford"; // Vehicle attribute
            public void honk() {  // Vehicle method
                System.out.println("Tuut, tuut!");
            }
            }

            class Car extends Vehicle {
            private String modelName = "Mustang";    // Car attribute
            public static void main(String[] args) {

                // Create a myCar object
                Car myCar = new Car();

                // Call the honk() method (from the Vehicle class) on the myCar object
                myCar.honk();

                // Display the value of the brand attribute (from the Vehicle class) and the value of the modelName from the Car class
                System.out.println(myCar.brand + " " + myCar.modelName);
            }
            }
Note: if you try to access a final class, this will generate an error.

Polymorphism

This means ‚Äúmany forms‚Äù, and this occurs when we have many classes that are related to each other by inheritance. As mentioned before, inheritance allows us to inherit methods and attributes from another class. Polymorphism uses those methods to perform different tasks. This allows us to perform a single action in different ways.
In this example, we have a superclass Animal that has a method called animalSound(). Subclasses of Animal could be Birds and Dogs, and they also have their own implementation of an animal sound. 


            class Animal {
            public void animalSound() {
                System.out.println("The animal makes a sound");
            }
            }

            class Bird extends  Animal {
            public  animalSound() {
                System.out.println("Chirps");
            }
            }

            class Dog extends Animal {
            public  animalSound() {
                System.out.println("Barks");
            }
            }

            class Main {
            public static void main(String[] args) {
                //Now we can create Bird and Dog objects and call the animalSound() method on both of them:
                    Animal myAnimal = new Animal();  // Create a Animal object
                Animal myBird = new Bird();  // Create a Bird object
                Animal myDog = new Dog();  // Create a Dog object
                myAnimal.animalSound();
                myBird.animalSound();
                myDog.animalSound();
            }
            }
